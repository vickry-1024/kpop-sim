## 每次工作的前置检查

1. **读取 [docs/04-开发执行步骤.md](docs/04-开发执行步骤.md)**，确认当前处于哪个开发阶段
2. **查看当天开发日志** `dev-logs/YYYY-MM-DD.md`（如有），了解上次进度
3. **明确本次任务**属于哪个阶段、哪些具体任务
4. **先读相关规范文档**再开始写代码（避免偏离标准）

## 开发日志更新规范

每天工作结束后必须更新开发日志，文件路径：`dev-logs/YYYY-MM-DD.md`

### 日志模板

```markdown
# 开发日志 - YYYY-MM-DD

## 今日完成
- [x] 完成项1
- [x] 完成项2

## 设计变更（如有）
- 变更内容 + 原因 + 影响范围

## 遇到的问题
- 问题描述 + 解决方案

## 待办事项
- [ ] 待办1
- [ ] 待办2

## 当前阶段
阶段X：[阶段名称] — 进度 X/总任务数
```

### 重要：设计变更同步规则

当用户对游戏设计或架构提出修改时：
1. **同步更新对应的设计文档**（[docs/01-游戏设计文档.md](docs/01-游戏设计文档.md) 或其他相关文档）
2. **在当日开发日志中记录变更**（变更内容+原因+影响范围）
3. **如果变更影响执行步骤**，同步更新 [docs/04-开发执行步骤.md](docs/04-开发执行步骤.md)

## 代码风格约定

### 必须遵守
- **注释用中文**，代码命名用英文
- **camelCase** 变量/函数，**PascalCase** 命名空间
- **2空格缩进**，必须加分号
- **CSS变量**管理所有颜色和间距（见 [docs/02-技术规范.md](docs/02-技术规范.md)）
- **移动端优先**：主适配320-430px，最大宽度480px

### 文件结构
- HTML：[index.html](index.html) 单页面，不新建HTML文件
- CSS：按功能分文件（reset/variables/main/components/phone/themes）
- JS：IIFE命名空间挂载到全局 `Game` 对象

### 关键约定
- 爱豆性别字段 `gender` 必须贯穿所有逻辑，不得硬编码性别代词
- 所有AI生成内容使用变量 `{他/她}` 而非写死
- 照片存 IndexedDB，游戏数据存 localStorage
