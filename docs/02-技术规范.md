# Kpop嫂子模拟器 — 技术规范

> 版本：v1.0 | 最后更新：2026-06-06

---

## 一、技术选型

| 层级 | 选型 | 原因 |
|------|------|------|
| **前端** | 纯 HTML/CSS/JS | 零依赖，GitHub Pages 直接部署 |
| **CSS** | 原生CSS + CSS变量 | 主题切换、无构建工具 |
| **JS** | ES6+ (原生) | 模块化用 IIFE 命名空间 |
| **AI服务** | DeepSeek API (Chat Completions) | 性价比最高 |
| **存储** | localStorage + IndexedDB | 纯前端，无后端 |
| **部署** | GitHub Pages | 免费、简单 |
| **版本管理** | Git + GitHub | 标准流程 |

## 二、项目文件结构

```
kpop-sim/
├── index.html              # 主入口，单页面应用
├── css/
│   ├── reset.css           # CSS Reset / Normalize
│   ├── variables.css       # CSS变量（颜色、字体、间距）
│   ├── main.css            # 主布局（五卡片导航、页面容器）
│   ├── components.css      # 组件样式（气泡、卡片、按钮、弹窗）
│   ├── phone.css           # 手机内页面样式
│   └── themes.css          # 主题切换（日常/秘密模式）
├── js/
│   ├── app.js              # 主入口，初始化
│   ├── router.js           # 页面路由/选项卡切换
│   ├── storage.js          # 本地存储（存档/读档）
│   ├── game-state.js       # 游戏状态管理（核心数据模型）
│   ├── character.js        # 角色创建逻辑
│   ├── turn.js             # 回合系统
│   ├── actions.js          # 行动系统
│   ├── events.js           # 事件系统（随机/固定/阈值）
│   ├── relationship.js     # 关系阶段管理
│   ├── phone/              # 手机子系统
│   │   ├── phone-main.js   # 手机主屏幕
│   │   ├── chat.js         # 聊天APP
│   │   ├── sns.js          # SNS
│   │   ├── news.js         # 新闻
│   │   └── call.js         # 通话
│   ├── api.js              # DeepSeek API 封装
│   ├── endings.js          # 结局系统
│   └── utils.js            # 工具函数
└── assets/
    └── icons/              # SVG图标（内联优先）
```

### 3.2 响应式断点

```css
/* 主适配：320px - 430px（移动端） */
/* 最大宽度限制：480px，超出居中显示 */
.app-container {
  max-width: 480px;
  margin: 0 auto;
  min-height: 100vh;
  min-height: 100dvh;
}
```

### 3.3 移动端适配

- `viewport: width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no`
- 安全区域适配：`safe-area-inset-*`
- 触摸最小点击区域：44×44px
- 所有交互元素使用 `touch-action: manipulation`

## 四、JavaScript规范

### 4.1 命名空间

```javascript
// 使用 IIFE 模块化，挂载到全局 Game 对象
const Game = (() => {
  // 私有变量和方法
  const _privateVar = null;

  // 公共API
  return {
    init() {},
    state: {},
    // ...
  };
})();
```

### 4.2 代码风格

- **语言**：代码用英文命名，注释用中文
- **命名**：camelCase（变量/函数）、PascalCase（命名空间）、UPPER_SNAKE（常量）
- **缩进**：2空格
- **引号**：单引号 `'`
- **分号**：必须

### 4.3 注释规范

```javascript
/**
 * 发送消息给指定爱豆
 * @param {string} idolId - 爱豆ID
 * @param {string} message - 消息内容
 * @returns {Promise<Object>} API返回的回复
 */
async function sendMessage(idolId, message) {
  // 构建请求体
  const body = { /* ... */ };
  // 调用API
  return await apiCall(body);
}
```

## 五、数据存储规范

### 5.1 localStorage Key命名

```
kpop-sim:save:slot1      # 存档槽1
kpop-sim:save:slot2      # 存档槽2
kpop-sim:save:slot3      # 存档槽3
kpop-sim:settings        # 游戏设置
kpop-sim:api-key         # DeepSeek API Key（加密存储）
```

### 5.2 IndexedDB 数据库

```
数据库名：kpop-sim-db
├── 表：photos           # 照片（头像/壁纸）
├── 表：chat-history     # 聊天记录
└── 表：event-log        # 事件日志
```

## 六、性能要求

| 指标 | 目标 |
|------|------|
| **首屏加载** | < 2秒（3G网络） |
| **页面切换** | < 300ms |
| **API响应等待** | 显示打字动画 |
| **图片大小** | 限制单张5MB，显示压缩 |
| **总包大小** | < 500KB（不含图片） |
