# Kpop嫂子模拟器 — 游戏设计文档

> 版本：v1.0 | 最后更新：2026-06-06

---

## 一、游戏概述

**游戏名称**：Kpop嫂子模拟器

**游戏简介**：一款以Kpop娱乐圈为背景的恋爱模拟网页游戏。玩家扮演自定义女主角，在回合制行程安排中与爱豆发展关系，经历攻略→恋爱→结婚三个阶段，面对现实压力和道德选择，最终走向不同的结局。

**核心体验**：沙盒式自由体验，无"正确"玩法——每个玩家的故事都由自己的选择塑造。

---

## 二、游戏类型与表现形式

| 维度 | 决定 |
|------|------|
| **游戏类型** | 回合制行程安排 + 选择驱动 + 多结局 |
| **平台** | 移动端网页（响应式，主适配320px-430px） |
| **UI框架** | 底部五卡片导航（日程/手机/我的/关系/事件） |
| **回合单位** | 半个月 |
| **总时长** | 2-3年（48-72回合），事件驱动结束 |
| **技术栈** | 纯 HTML/CSS/JS + DeepSeek API |

---

## 三、核心基调与目标

| 维度 | 决定 |
|------|------|
| **核心哲学** | 提供有意义的选择，不引导"完美路线" |
| **胜负标准** | ❌ 无固定标准，所有结局都是"你的故事" |
| **基调** | 玩家自主塑造——可甜可虐可抓马，后果取决于选择 |
| **关系阶段** | 攻略期（可多人）→ 恋爱期（单人，玩家可出轨）→ 结婚期 |

## 四、女主人设

| 维度 | 决定 |
|------|------|
| **姓名** | 玩家自由输入 |
| **身份** | 预设标签可选 + 自定义输入 |
| **性格/人设** | 预设标签可选 + 自定义输入 |

### 身份标签预设
素人粉丝 / 练习生 / 造型师 / 记者 / 翻译 / 站姐 / 综艺PD / 青梅竹马 / 学生 / 化妆师 / 运动教练 / 名门社交好友 / 其他（自定义）

### 性格标签预设
直球勇敢 / 温柔体贴 / 傲娇高冷 / 腹黑心机 / 社恐慢热 / 其他（自定义）

### 身份对数值的影响
不同身份影响好感度加成和嫌疑度判定：
- 粉丝/站姐：追星场景好感+，但被拍到→嫌疑度暴涨
- 练习生/同行：工作场景好感+，同框合理→嫌疑度低
- 青梅竹马/同学：天然信任基础→初期嫌疑度极低
- 工作人员：专业场景好感+，合理接触多→嫌疑度低
- 名门/社交圈：社交场合合理→嫌疑度看场合

## 五、爱豆人设

| 维度 | 决定 |
|------|------|
| **设计方式** | 原创Kpop风爱豆 + 玩家完全自定义 |
| **姓名** | 自由输入 |
| **性别** | 自由选择（男/女） |
| **性格/人设** | 预设标签可选 + 自行输入描述 |
| **所属团体** | 自由输入团名 |
| **人数** | 玩家自行决定攻略几个爱豆 |
| **备注/爱称** | 可设置，系统使用原名 |

### ⚠️ 性别约束
所有后续剧情/对话/事件/AI生成内容必须严格匹配玩家设定的性别：
- 数据模型：每个爱豆存储 `gender` 字段
- AI对话：系统提示词明确传入性别，使用正确代词
- 事件文本：使用变量（如`{他/她}`、`{哥哥/姐姐}`）而非写死
- 不得出现"设定女爱豆但剧情套用男爱豆模板"的情况

## 六、核心数值系统

| 数值 | 范围 | 说明 |
|------|------|------|
| ❤️ 好感度 | 0-100/每人 | 分爱豆独立计算，受身份标签加成/减成 |
| 😰 压力值 | 0-100 | 过高触发负面事件（崩溃、失误） |
| 🕵️ 嫌疑度 | 0-100 | 恋情暴露风险，不同身份下判定不同 |
| ⚡ 体力 | 0-100 | 每回合行动消耗，休息恢复 |
| 💎 魅力 | 0-100 | 影响初始好感、事件触发门槛、对话成功率 |

### 显示方式
所有数值**全可见**，以进度条/数字形式展示。

## 七、主要玩法模块

### 7.1 回合制核心循环

```
每回合开始
  ├── 📋 查看当前数值面板
  ├── 🎯 选择行动（消耗体力）
  │     ├── 💬 和爱豆互动（发消息/约会/探班/…）
  │     ├── 🏋️ 提升自己（健身/学才艺/护肤/…）
  │     ├── 📱 经营社交（发SNS/和粉丝互动/…）
  │     ├── 💼 工作/学业
  │     ├── 🎁 送礼/应援
  │     ├── 🔍 收集情报
  │     └── 🏥 休息（恢复体力、降压）
  ├── 🎲 触发事件（如有）
  ├── 📊 数值变动结算
  └── 📖 推进剧情（如达到条件）
```

### 7.2 行动系统
- 不同身份/人设解锁不同行动子选项
- 每个行动消耗体力，不同行动消耗量不同
- 不同选项影响不同数值

### 7.3 底部五卡片导航

| 选项卡 | 功能 |
|--------|------|
| 📅 日程 | 每回合的行动安排 |
| 📱 手机 | 核心社交工具（聊天/SNS/新闻/通话/购物） |
| 👤 我的 | 查看女主数值、身份、属性面板 |
| 💜 关系 | 查看各爱豆好感度、关系状态、嫌疑度 |
| 🗂️ 事件 | 已触发事件回顾、剧情日志、结局回顾 |

## 八、手机系统

### 8.1 手机内软件

| 软件 | 功能 | 互动对象 |
|------|------|---------|
| 💬 聊天APP | 私聊 | 一对一（和爱豆） |
| 📞 通话 | 语音通话（重要/紧急） | 一对一 |
| 📸 SNS/社交媒体 | 发动态、看动态、互动 | 公开/半公开 |
| 📰 新闻/热搜 | 娱乐新闻、绯闻、回归消息 | 被动接收 |
| 🛒 购物/应援 | 买礼物、应援物、服饰 | 系统交互 |
| 📧 邮件 | 工作/学校通知 | 系统/工作 |

### 8.2 多手机机制
- **主手机**：日常使用，所有联系人可见
- **秘密手机**：剧情触发后获取（经纪人给/自己买）
- 切换方式：手机顶部标签或下拉菜单
- 限制：最多2-3台手机
- 风险：秘密手机被发现→嫌疑度暴涨
- 视觉区分：主手机亮色UI / 秘密手机暗色UI

### 8.3 对话系统
- 预设选项（2-4个）+ 自定义输入框
- 爱豆聊天头像可上传，未上传则不显示（仅气泡）
- 玩家手机壁纸可自定义
- 对话历史本地存储

## 九、剧情框架与结局

### 9.1 驱动方式
时间 + 关系 + 事件 混合驱动

### 9.2 主线故事
❌ 无——纯沙盒，自由体验

### 9.3 结局（约15个）

| 维度 | 结局方向 |
|------|---------|
| 💒 结婚 | 和爱豆A公开结婚 / 和爱豆A秘密结婚 / 和爱豆B公开结婚…… |
| 💔 分手/离婚 | 出轨被发现→分手 / 婚后出轨→离婚 / 压力→关系破裂 |
| 🌟 事业巅峰 | 事业爱情双收 / 为事业放弃感情 |
| 🔥 曝光 | 媒体曝光→被迫公开 / 公司拆散 / 粉丝抵制 |
| 😈 修罗场 | 出轨全身而退 / 多线操作被抓→身败名裂 |
| 🏃 独自美丽 | 谁都没选→专注自己 / 退圈回普通生活 |
| 🎭 意外结局 | 爱上计划外的人 / 双双退圈私奔 / 开放式关系 |

### 9.4 事件系统（三层触发）

| 类型 | 触发方式 | 示例 |
|------|---------|------|
| 🎪 固定事件 | 时间/日程触发 | 回归期、颁奖礼、生日、纪念日 |
| 🎲 随机事件 | 概率触发 | 偶遇、突发新闻、名场面 |
| 📈 数值触发 | 阈值触发 | 好感50→约会 / 嫌疑80→被质问 / 好感100→告白 |
