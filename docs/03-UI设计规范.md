# Kpop嫂子模拟器 — UI设计规范

> 版本：v1.0 | 最后更新：2026-06-06

---

## 一、设计方向

### 主方案：D. 动态氛围风

**核心理念**：UI随游戏数值变化，像在"呼吸"一样。日常温馨、秘密紧张、数值不同色调不同。

### 备选方案：A. 韩系极简风

**核心理念**：大量留白、圆角卡片、轻阴影，像KakaoTalk/Instagram一样的干净体验。

> 优先实现D，如遇到技术困难则降级为A。

## 二、色彩系统（方案D）

### 2.1 日常模式（默认）

| 色值 | 用途 |
|------|------|
| `#FFFAFA` | 主背景（奶油白） |
| `#FFFFFF` | 卡片背景 |
| `#F8A5C2` | 主色调（柔粉） |
| `#FCE4EC` | 浅粉强调 |
| `#2D2D2D` | 主文字色 |
| `#8E8E93` | 次要文字色 |

### 2.2 秘密模式

| 色值 | 用途 |
|------|------|
| `#1A1A2E` | 主背景（深色） |
| `#16213E` | 卡片背景 |
| `#BB86FC` | 强调色（淡紫） |
| `#CF6679` | 警告色 |
| `#E0E0E0` | 主文字色 |

### 2.3 数值联动色

| 数值状态 | 视觉变化 |
|----------|---------|
| 好感度 > 80 | 聊天气泡带微粉色光晕（`box-shadow: 0 0 8px rgba(248,165,194,0.4)`） |
| 嫌疑度 > 60 | 屏幕顶部边缘出现微红渐变（`border-top: 2px solid #FF4757`） |
| 嫌疑度 > 80 | 红色脉冲动画 |
| 压力值 > 70 | 整体色调偏灰（`filter: saturate(0.7)`） |
| 压力值 > 90 | 屏幕偶尔轻微抖动（CSS animation） |

### 2.4 特殊时刻色

| 时刻 | 色调 |
|------|------|
| 告白/确认关系 | 柔粉渐变背景过渡 |
| 分手/离婚 | 冷灰蓝过渡 |
| D社曝光 | 红色闪烁警告 |

## 四、组件规范

### 4.1 底部导航栏

```
┌──────────────────────────────┐
│  内容区域                     │
│                              │
├────────┬────────┬────────┬───┤
│  📅    │  📱    │  👤    │💜 │  ← 五卡片（图标+文字标签）
│ 日程   │ 手机   │ 我的    │关系│     高度：56px + safe-area
└────────┴────────┴────────┴───┘
```

- 5个选项卡图标等宽分布
- 当前选中：主色调高亮
- 未选中：灰色 `#C7C7CC`
- 激活时有微弹跳动画
- 视觉区分：游戏UI层和手机内UI层

### 4.2 卡片

```css
.card {
  background: var(--bg-card);
  border-radius: var(--radius-md);  /* 12px */
  padding: var(--spacing-md);       /* 16px */
  box-shadow: var(--shadow-card);
  margin-bottom: var(--spacing-sm);  /* 8px */
}
```

### 4.3 消息气泡

```
┌─────────────────────┐
│ [头像] 消息内容...    │  ← 对方消息（左对齐，浅灰底）
└─────────────────────┘

        ┌─────────────────────┐
        │ 消息内容...          │  ← 玩家消息（右对齐，柔粉底）
        └─────────────────────┘
```

- 无头像时不显示头像占位，仅气泡
- 好感度>80时气泡带粉光晕
- 气泡最大宽度：75%
- 圆角：16px，自己的消息右下角稍小

### 4.4 按钮

```css
/* 主按钮 */
.btn-primary {
  background: var(--color-primary);
  color: #FFF;
  border-radius: 25px;     /* 胶囊形 */
  padding: 12px 24px;
  font-size: 15px;
  min-height: 44px;        /* 触控最小尺寸 */
}

/* 次要按钮 */
.btn-secondary {
  background: transparent;
  border: 1px solid var(--color-primary);
  color: var(--color-primary);
  border-radius: 25px;
  padding: 12px 24px;
}

/* 危险按钮 */
.btn-danger {
  background: var(--color-danger);
  color: #FFF;
  border-radius: 25px;
  padding: 12px 24px;
}
```

### 4.5 进度条

```css
.progress-bar {
  height: 6px;
  border-radius: 3px;
  background: #E0E0E0;
  overflow: hidden;
}
.progress-bar-fill {
  height: 100%;
  border-radius: 3px;
  transition: width 0.5s ease;
}
```

- 好感度：粉色填充
- 压力值：灰色→红色（50以上渐变）
- 嫌疑度：黄色→红色（60以上）
- 体力：绿色→黄色→红色（递减）
- 魅力：紫色填充

### 4.6 弹窗/模态

```css
.modal-overlay {
  background: rgba(0,0,0,0.5);
  display: flex;
  align-items: center;
  justify-content: center;
}
.modal-content {
  background: #FFF;
  border-radius: 20px;
  padding: 24px;
  max-width: 90vw;
  max-height: 80vh;
  overflow-y: auto;
}
```

## 五、手机内UI设计

### 5.1 手机主屏幕

模拟真实手机桌面：
- 上方状态栏（时间、电量、信号图标——装饰性）
- 应用图标网格（3列或4列）
- 每个APP有图标+名称标签
- 可设置壁纸背景

### 5.2 聊天APP（参照KakaoTalk）

- 联系人列表：头像+姓名+最新消息预览
- 聊天界面：气泡式对话
- 底部输入栏：输入框 + 发送按钮
- 预设选项以"快捷回复"形式呈现

### 5.3 SNS（参照Instagram/Twitter）

- 动态流：图片卡片+文字+互动按钮
- "发布"按钮
- 发布界面：文字输入+图片选择

### 5.4 新闻（参照Naver新闻）

- 标题列表
- 点击展开详情
- 娱乐分类优先

### 5.5 通话界面

- 来电：全屏显示，接听/挂断按钮
- 去电：等待连接动画
- 通话历史列表

## 六、动画规范

| 动画 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 页面切换 | 250ms | ease-out | 滑入/淡入 |
| 选项卡切换 | 150ms | ease | 图标弹跳 |
| 消息出现 | 200ms | ease-out | 从下向上滑入 |
| 数值变化 | 500ms | ease | 进度条过渡 |
| 弹窗 | 200ms | ease-out | 缩放+淡入 |
| 色调切换 | 800ms | ease-in-out | 渐变过渡 |
| 心跳/脉冲 | 1.5s | ease-in-out | 循环 |

### 动效原则
- 不卡顿优先：所有动画使用 `transform` 和 `opacity`（GPU加速）
- 尊重用户偏好：检测 `prefers-reduced-motion`
- 不要过度动效，每屏最多1-2个同时进行的动画
